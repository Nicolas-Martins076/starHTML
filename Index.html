Estrutura Básica do HTML

Tags Explicação de cada Tag
<html> </html> A tag HTML é a raiz de um documento HTML. Ela envolve todo o conteúdo da página e define o início e o fim do documento.
<head></head> A seção head contém informações importantes para o navegador, mas não visíveis para o usuário. Isso inclui tags como title, que define o título da página exibido na aba do navegador.
<body></body> A seção body contém o conteúdo visível da página que será exibido para o usuário, como texto, imagens, links, etc.

DIVIDIDO UMA PAGINA EM ESTRUTURAS DE ELEMENTOS:


  1. ASIDE: O elemento aside é usado para conteúdo tangencial ou relacionado ao conteúdo principal da página, mas que pode ser considerado separado dele. Geralmente, é utilizado para informações adicionais, como anúncios, notas de rodapé, links relacionados, etc.

  2. SECTION: O elemento section é utilizado para agrupar conteúdo tematicamente relacionado em uma página. Ele ajuda a organizar o conteúdo em partes distintas, tornando mais fácil para os desenvolvedores e leitores entenderem a estrutura da página.

  3. NAV: O elemento nav é usado para definir uma seção de navegação em uma página. É comumente usado para agrupar links de navegação, como menus, listas de links para outras páginas do site, índices, tabelas de conteúdo, etc.

  4. FOOTER: O elemento footer é utilizado para representar informações de rodapé em uma página. Geralmente, é utilizado para incluir informações como direitos autorais, informações de contato, links para políticas de privacidade ou termos de uso, etc. O footer é geralmente colocado no final da página, mas pode ser usado dentro de outras seções, como article ou section.

  5. MAIN: O elemento main é usado para envolver o conteúdo principal de uma página HTML. Ele representa o conteúdo central ou principal da página, excluindo cabeçalhos, rodapés, barras laterais e outros conteúdos que são repetitivos em várias páginas do site.

    6. Esses elementos são importantes para a estruturação e organização de uma página web, tornando o código mais semântico e compreensível, tanto para os desenvolvedores quanto para os navegadores e leitores de tela.
